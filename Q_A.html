Q&A:

  • What are GTFS files?
  • The General Transit Feed Specification (GTFS) is an Open Standard used to distribute relevant information about transit systems to riders. It allows public transit agencies to publish their transit data in a format that can be consumed by a wide variety of software applications. Today, the GTFS data format is used by thousands of public transport providers. For more information, please visit the link.
  • Where can I find GTFS files?
  • You can find GTFS files from the TransitFeeds website by entering your city's name in the search bar.
  • Why don't we assign each charging station a power node?
  • In reality, the distance between two neighboring bus stops is often considerably shorter than the distance between two nodes in the power grid. Therefore, it's necessary to refine our approach to selecting power grid nodes from the existing transportation nodes. Additionally, adjacent charging stations can be efficiently allocated to their nearest power node for energy support. This ensures a more practical and effective distribution of power resources.
  • What should be done if no network is generated?
  • If no network is generated, please check the following:
    • Make sure the GTFS file is in the correct format.
    • Make sure the GTFS file contains "routes.txt", "trips.txt", "shapes.txt", "stops.txt", and "stop_times.txt".
    • Ensure that the "stop_times.txt" file in your GTFS file has the "shape_dist_traveled" columns.
    • Review your input parameters - 'dist_threshold', 'common_stop_threshold', and 'threshold_power_node' - to make sure they're suitable for the current bus routes.
